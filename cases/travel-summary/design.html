Дизайн-решения

Дизайн решения, основанные на данных исследования. Центральный вопрос: как сделать так, чтобы пользователь захотел поделиться.

Точка входа: карточка на главном экране

Карточка появляется через 24–48 часов после поездки и показывает тизер с забавным фактом ещё до перехода в карусель.

Почему именно так:

  • 19 из 27 (70%) согласились: забавные факты снижают страх выглядеть хвастливо
  • Нейтральный анонс «итог готов» не даёт причины кликнуть, особенно Сегменту 2
  • Тизер переключает мотивацию с «надо посмотреть» на «интересно, что там ещё»

Карусель из 5 карточек

Порядок карточек выстроен по логике нарратива: эмоциональный hook → данные → призыв к действию. Жёлтый чип с забавным фактом присутствует на каждой информационной карточке.

Структура карусели:

  • Слайд 1/5: Обложка: полноэкранное фото, город, даты, три стат-чипа (дни / км / рубли)
  • Слайд 2/5: Маршрут: схематичная карта, пронумерованные точки, жёлтый чип с фактом о дистанции
  • Слайд 3/5: Места: список карточек с фото, названием и временем пребывания, чип про еду поездки
  • Слайд 4/5: Бюджет: общая сумма, разбивка по категориям, сравнение с прошлой поездкой
  • Слайд 5/5: Итог: achievement-чип, три метрики, иерархия CTA

Экран редактирования

Три секции в строго определённом порядке: ПРИВАТНОСТЬ (тогглы скрытия), ФОТОГРАФИИ (замена обложки), МЕСТА В МАРШРУТЕ (удаление).

Ключевые решения:

  • ПРИВАТНОСТЬ идёт первой: 18 из 27 (67%) назвали страх раскрытия финансов главным барьером
  • Тоггл «Скрыть расходы» в состоянии OFF по умолчанию
  • Добавление пропущенных мест, после MVP, потому что требует поиска, интеграции с картами и базой туристических мест

Раздел «Мои поездки»

Двухколоночная сетка карточек с фотографиями, сгруппированная по годам. На каждой карточке: чип с типом содержимого: История / Статистика / Маршрут / Сравнение.

Зачем чипы:

  • Дают пользователю превью содержимого без открытия
  • Снижают порог повторного открытия через 6–12 месяцев
  • Выбор чипа: самый уникальный элемент конкретного summary, а не дата

MVP: минимальный состав

Элемент Обоснование
Генерация summary через 24–48ч с забавными фактами Оптимальный тайминг + снятие барьера шеринга
Полное редактирование (скрытие данных, замена фото, корректировка текста) 16 из 27 назвали критически важным
Конкретные названия мест и количественные метрики 21 из 27 оценили как ключевую ценность
Карточка на главном экране с превью факта Точка входа с высоким CTR: hook до открытия summary

Сравнение поездок для пользователей с 2+ 9 из 15 готовы регулярно возвращаться к сравнению
Напоминания раз в 6–12 месяцев Долгосрочный retention при низком раздражении
AI-генерация альтернативных формулировок фактов Снижает барьер edit, помогает Сегменту 1
