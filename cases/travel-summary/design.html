Дизайн

User flow (swim-lanes, 6 фаз), UI kit на базе дизайн-системы Яндекс Путешествий и 10+ экранов с аннотациями, привязанными к данным исследования. Центральный вопрос: как сделать так, чтобы пользователь захотел поделиться.

User Flow

Полный жизненный цикл travel summary: от момента бронирования до повторного возврата через год. Три дорожки: пользователь, система, трекинг. Шесть фаз.

Ключевые ветвления в диаграмме:

  • Достаточно ли данных для генерации summary?
  • Пользователь дал согласие на геолокацию и фото?
  • Открыть сейчас или отложить?
  • Редактировать или делиться напрямую?
  • Есть ли подписка Плюс для убирания вотермарки?

Точка входа: карточка на главном экране

Карточка появляется через 24–48 часов после поездки и показывает тизер с забавным фактом ещё до перехода в карусель. Если тизер работает на уровне карточки, он создаёт hook в точке входа.

Почему именно так:

  • 19 из 27 (70%) согласились: забавные факты снижают страх выглядеть хвастливо
  • Нейтральный анонс «итог готов» не даёт причины кликнуть, особенно Сегменту 2
  • Тизер переключает мотивацию с «надо посмотреть» на «интересно, что там ещё»

Карусель из 5 карточек

Порядок карточек выстроен по логике нарратива: эмоциональный hook → данные → призыв к действию. Жёлтый чип с забавным фактом присутствует на каждой информационной карточке.

Структура карусели:

  • Слайд 1/5: Обложка: full-bleed фото, город, даты, три стат-чипа (дни / км / рубли)
  • Слайд 2/5: Маршрут: схематичная карта, пронумерованные точки, жёлтый чип с фактом о дистанции
  • Слайд 3/5: Места: список карточек с фото, названием и временем пребывания, чип про еду поездки
  • Слайд 4/5: Бюджет: общая сумма, разбивка по категориям, сравнение с прошлой поездкой
  • Слайд 5/5: Итог: achievement-чип, три метрики, иерархия CTA

Экран редактирования

Три секции в строго определённом порядке: ПРИВАТНОСТЬ (тогглы скрытия), ФОТОГРАФИИ (замена обложки), МЕСТА В МАРШРУТЕ (удаление). Порядок секций: это продуктовое решение, а не вёрстка.

Ключевые решения:

  • ПРИВАТНОСТЬ идёт первой: 18 из 27 (67%) назвали страх раскрытия финансов главным барьером
  • Тоггл «Скрыть расходы» в состоянии ON по умолчанию: deliberate default
  • Добавление пропущенных мест, за MVP: требует поиска и интеграции с базой мест

Раздел «Мои поездки»

Двухколоночная сетка карточек с фотографиями, сгруппированная по годам. На каждой карточке: чип с типом содержимого: История / Статистика / Маршрут / Сравнение.

Зачем чипы:

  • Дают пользователю превью содержимого без открытия: паттерн ambient awareness
  • Снижают порог повторного открытия через 6–12 месяцев
  • Выбор чипа: самый уникальный элемент конкретного summary, а не дата
