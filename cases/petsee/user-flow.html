User Flow

Два параллельных потока — владелец и ситтер. Оба начинаются с единой точки входа (онбординг), расходятся по роли и сходятся в момент активной передержки.

Схема User Flow — диаграмма

Полная диаграмма в Figma с развилками, критическими сценариями и экранными переходами

Общий старт

Онбординг → Выбор роли

Запуск приложения
└── Онбординг (контент адаптируется после выбора роли)
     ├── Пользовательское соглашение
     ├── Политика обработки данных
     └── Регистрация по номеру телефона
          └── Развилка: Владелец / Ситтер

Поток: Владелец питомца

1. Создание профиля

Карточка питомца
├── Основное (имя, вид, порода, возраст, фото)
├── Подробнее (характер, триггеры, реакции)
├── Здоровье (болезни, лекарства, контакт ветклиники)
├── Дополнительно (режим дня, игрушки, инструкции)
└── Способ оплаты → Свободный адрес владельца

2. Поиск

Главная страница поиска
├── Выбор услуги → Выбор питомца (из профиля)
└── Результаты поиска
     ├── Фильтр «Проверен PetSee»
     ├── Дополнительные фильтры
     ├── Просмотр всех отзывов
     └── ⚠ Подбор ситтеров не найден — экран-заглушка

3. Бронирование

Анкета ситтера
└── Бронирование
     ├── Выбор услуги → Проверка условий → Состав цены
     └── Instant Book?
          ├── Да → Автоприём → Создание оплаты ситтеру
          └── Нет → Ожидание ответа ситтера → Способ оплаты

4–5. Передержка и завершение

Бронирование подтверждено
└── Чат → Запрос отчёта

Завершение передержки
├── Перечисление средств ситтеру
├── Оставить отзыв
└── Пригласить друзей (реферальная программа)

Поток: Ситтер

1. Создание профиля

Анкета ситтера
├── Основное (фото, имя, локация, о себе)
├── Услуги (виды, цены, бейдж «Высокооплачиваемая услуга»)
├── Подробности (условия, опыт)
├── Опыт (животные, которых принимает)
└── Способ оплаты
     ├── Верификация ← → Выход
     └── Теория PetSee ← → Выход
          └── Профиль ситтера → Свободный ситтер

2. Принятие заявки

Пуш-уведомление
└── Instant Book?
     ├── Да → Автоприём
     └── Нет → Список заявок
               └── Просмотр карточки питомца
                    └── Принять (кнопка или свайп вправо) / Отклонить

3–4. Передержка и завершение

Бронирование подтверждено
└── Чат
     ├── Пуш «Отправь отчёт» → камера напрямую
     └── Шаблоны быстрых ответов

Завершение передержки
├── Перечисление средств
└── Принять другие заказы
